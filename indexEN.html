Project 12

  • GitHub
last update : 9th June 2021

Object

Create an application with a social impact (in the sense that the application must be able to be useful to others).
The application model must be fully tested.

Principle. As part of this project, I chose to create an application motivating the user to perform sports activities.
The health situation in 2021 and the resulting confinement are causing the least motivated people to lack sports activities. It can have negative repercussions on physical health.
The purpose of this app is to give these people motivation (additional to staying healthy).
Thanks to this application, the user will be able to allow himself to spend a little of his money to have fun.
The application will in fact award the user points for each physical activity performed.
These points are then used to obtain the amount that the user can afford to spend out of their own money thanks to the physical efforts that they have put in.
The user can obviously configure at his convenience the number of points necessary to obtain an euro according to his budget. Thus if the user finds that the amounts obtained are too high given his budget, he can readjust the number of points necessary in order to reduce the amounts obtained subsequently (and vice versa).

Technical side. My training having been focused on the use of UIKit, I decided to initiate myself to SwiftUI in order to finish this training by having carried out applications using both.

Skills

  • SwiftUI ;
  • MVVM.

notre histoire

  • GitHub
last update : 7th July 2021

Object

Making my marriage proposal thanks to a video game retracing our life as a couple.

Principle. This application is the one that matters the most to me because it allowed me to ask my wife to marry me. It is also the first video game that I developed, at the very beginning of my training and at the same time.

This is a platform game ( Super Mario type ). The game is divided into 11 levels, of which 10 are game levels and the last one allows you to make your request.

Each of the first 10 levels represents a period in the life of the couple.
In each of these levels, the goal of the main character (representing the lucky choosen one) is to find 5 hidden objects (each representing a particular memory of the period concerned by the level). These 5 items, once collected, allow you to open the cage in which the other character is locked (the person making the proposal).

The eleventh level represents the marriage proposal, and should only be played when the game creator is ready to make the request. Indeed, once the first 10 levels have been completed, an alert message will be displayed on the game's home screen to indicate the fact that a (false) bug has occurred.
If the creator of the game is not yet ready to make his request, he can procrastinate by saying that he will take a look at the code to fix the bug.
When he's ready to make his request, all he has to do is press the moving heart on the home screen until it turns white. When the heart is white, it means level 11 can be played.

In level 11, the main character must move forward until he comes face to face with the character of the person who is going to ask. This character then announces that this level is to be played in pairs, and that the creator of the game will explain to him (in real life, outside the game) how this level is played. The person then makes a small introduction to his request. Once the introduction is completed, the lucky choosen one can then press on the screen, and the screen then displays that it is the marriage proposal. All you have to do then is bend your knee and hope for a positive response ...

Technical side. The game was made thanks to SpriteKit and GameplayKit. As stated, it was created when I had just started my training, so I would have to improve it with my knowledge gained since then, clean up the code to make it more readable, but it works fine in the state. At the moment, I am writing a README file in order to indicate how to customize it.

Skills

  • GameplayKit ;
  • SpriteKit.
